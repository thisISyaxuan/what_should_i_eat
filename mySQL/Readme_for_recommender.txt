1. 基於內容推薦(Content-based Recommendation)
    (1) 根據 (餐廳與自己初始所建立的喜好相似度) 產生推薦結果
        透過每間餐廳的標籤與使用者註冊後的初始喜好產生餘弦相似矩陣與相似排序

    (2) 根據 (自己之前吃過的餐廳喜好) 產生推薦結果
        在計帳資料庫中，透過每間餐廳的標籤與自己的計帳紀錄產生餘弦相似矩陣與相似排序

2. 基於使用者的協同過濾(User-based Collaborative Filtering)
    (1) 根據 (共同喜好的其他使用者) 產生推薦結果
        在計帳資料庫中，找到與使用者與自己喜好相近的其他使用者，根據這些使用者紀錄過的帳與評分，推薦評分最高且使用者尚未吃過的餐廳

--------------------------------------------

# 過濾使用者需求 -> 產生推薦結果 -> 根據要求排序

# 過濾
    餐廳是否營業中
    餐廳是否提供正餐
    指定類別
# 推薦
    if 沒有記帳紀錄的使用者:
        output = 1-(1) 的推薦結果
    else:
        output = 三個演算法的集合 (各推薦 1/3 的餐廳數)
# 排序
    預設
    餐廳在 google map 的評分由高到低
    餐廳距離使用者由盡到遠

# e.g.
    # input(rID=1, 推薦結果的餐廳數=15, 在 2-(1) 要比較的使用者人數=10)
    # 是已經有計帳紀錄的使用者
    # 結果需要根據評分由高到低顯示正在營業的餐廳
    
    1. 先篩選出證在營業中的餐廳
    2. 執行三個演算法, 各產生 15/3 = 5 間餐廳
    3. 找出三個演算法結果的集合, 若有缺則補
    4. 將 15 間餐廳依據評分排序
    5. 回傳結果給前端
